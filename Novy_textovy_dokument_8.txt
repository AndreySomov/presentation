1. What is machine learning?

Мир полон задач, которые для нас, носителей естественного интеллекта, не представляют особых трудностей. Например, определить, кто изображен на фото кошка или собака? Какой возраст примерно имеет человек?
Эти задачи имеют сложные неявные зависимости, в отличие от перевода минут в часы или расчета траектории полета метеорита, которые иногда имеют сложные, но очевидные зависимости.


Если мы хотим научить машину решать задачи, для которых формулы не существуют ни в одном учебнике мира, то мы не можем обойтись без машинного обучения.



Машинное обучение предполагает, что компьютер распознает шаблоны, используя примеры, а не программируется с использованием определенных правил. Подразумвается, что все необходимые шаблоны и зависимости уже находятся в данных.


3. How do machines learn?
Давайте на конкретном примере разберем как применить машинное обучение.

представте, что наша компания продает роботов-убийц. У нас есть свой собственный сайт, где люди оставляют отзывы, но мы хотим понять, что люди пишут о нашем товаре в социальных сетях.
Для этого нам нужно:
1. Собрать данные для обучающей выборки: каждый комментарий мы обозначим как 'x' а его окрас как 'y'(положительный или отрицательный);
2. Далее на нужно разработать нашу модель, котрую мы будем применять к нашим данным, назовем ее a(x), и обучить ее.
3. Применить нашу модель для комментариев из социальных сетей, в которых нас упоминают.

Чем более точные будут данные, тем эффективнее будет наш алгоритм a(x). 
В нашем случаи в качестве обучающей выборки подойдут комментарии с нашего сайта, где пользователи помимо текста ставят еще и оценку нашему товару и сервису.
Каждый комментарий пользователя мы будет обозначать за 'x' его тональность за 'y' (+(положительный) или -(отрицательный));




Для примера возьмем 2 комментария с ярковыроженными тональностями.


Thank you very much! This is exactly what I was looking for, the quality is excellent! My neighbors will no longer bother me. The only negative is that he burned the floor of the area, with my house too.


Awful quality! All the Gypsies of the camp were thrown off to him, but he does not understand that we order him, I advise you to avoid this manufacturer. Money back and thanks for that!



Нам придется упростить задачу. Давайте с вами напишем, так называемый, мешок слов. Это 1000 наиболее важных по нашему мнению слов, способных охарактеризовать комментарий:


 1. большое     (1, 0, 0, 0, 0, 0, ... 0)
 2. спасибо     (0, 1, 0, 0, 0, 0, ... 0)
 3. отличное    (0, 0, 1, 0, 0, 0, ... 0)
 4. сломался    (0, 0, 0, 1, 0, 0, ... 0)
 5. избегать    (0, 0, 0, 0, 1, 0, ... 0)
 6. ужасное	(0, 0, 0, 0, 0, 1, ... 0)
 5. ...
 1000. минус    (0, 0, 0, 0, 0, 0, ... 1)

Каждому слову будет соответствовать своя кодировка из 1000 чисел.
Найдем все слова в тексте, сложим их кодировки, и получим кодировку нашего комментария состоящую так же из 1000 чисел

Первый отзыв (1(большое), 1(спасибо), 1(отличное), 0, 0, 0, ... 1(минус)) 
Второй отзыв (0, 1(спасибо), 0, 1(сломался), 1(избегать), 1(ужасное) ... 0))

Затем нам необходимо применить веса к нашим селекторам: 


a(x) = 2*(большое) + 4*(спасибо) + 5*(отличное) - 3*(минус) =  8

a(x) = 2*(спасибо) - 5*(сломался) - 4*(избегать) - 5*(ужасное) = -10



Отбрасываем число и берем знак получая + или - на выходе, это и будет характеризовать результат работы нашей модели как + или -.

Но откуда мы знаем что веса при наших селектора верные? Почему слово 'спасибо' равно по весу слову 'ужасный'. ВВзгляните на этот комментарий:

Мой старый тостер сломался, знакомый посоветовал этого производителя. Пользуюсь уже 2 месяца полет нормальный.


Здесь слово 'сломался' не имеет негативного окраса. А камментарий в целом нейтральный.

Для того чтобы минимизировать процент ошибок в машинном обучении существуют функция потерь и оптимизатор.

Функция потерь сравнит результат на выходе нашей модели с реальной оценкой пользователя и и результатом ее работы будет мера успеха в решении стоящей задачи. Важно понимать что не существует универсальной функции потерь, под каждую задачу нужно выбрать подходящую, или написать ее самому если задача специфическая.

Аглоритм обучения(оптимизатор) определяет как именно будет меняется модель, на основании результата функции потерь. В нашем случаи мы будем менять веса наших селекторов.

Так и происходит машинное обучение, мы прогоняем необходимые данные через нашу модель затем применяем функцию потерь и оптимизатор, до тех пор пока не получим необходимую нам точность на выходе модели.

Далее мы можем достать комментарии из социальных сетей которые уже не имеют конкретную оценку пользователя, и применить нашу модель к ним.

Типы машинного обучения:

Supervised learning (обучение с учителем): В примере выше мы использовали данный подход. Мы подгоняем нашу модель так, чтобы ее результаты совпадали с результатами обучающей выборки.

Unsupervised learning (обучение без учителя): Машина учится не по выбранным данным. Смысл в том, что среди данных нет «правильного» ответа, машина должна найти зависимость между объектами и разбить их на группы. В нашем примере мы бы сразу взяли комментарии из социальных сетей, прогнали через модель а на выходе получили бы группы комментариев с определенными признаками, затем нам бы нужно было сообщить модели какие из них относить к положительным, а какие к отрицательным.

Reinforcement learning (обучение с подкреплением): Машина учится через систему на основе вознаграждения. Например при обучении алгоритма игры в шахмат. Когда он побеждает 
 мы относим это к желаемым события, а когда проигрывает наоборот. Это позволяет алгоритму понять какие паттерны успешные а какие нет.

2. Применение машинного обучения:



Обзор того, где вы можете использовать машинное обучение:

Транспорт: автопилоты в автомобилях.
Здравоохранение: прогнозирование диагнозов пациентов для врачей
Социальная сеть: прогнозирование совместимости знакомств
Финансы: прогнозирование мошенничества с кредитными картами
Электронная коммерция: прогнозирование оттока клиентов, рекомендации товаров
Биология: поиск паттернов генных мутаций, которые могут вызвать рак
